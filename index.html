KlipperVisuals

Interactive visualizations of motion-control internals in Klipper / Kalico — pressure advance, input shaping, and resonance compensation.

Nonlinear pressure advance explorertanh vs recipr

Compare the two nonlinear pressure-advance models. Adjust ADVANCE, OFFSET, VELOCITY and a test move to see advance-vs-velocity, extruder flow rate, and acceleration demand, with the firmware's parabolic extruder smoother and a time-offset control.

Input shaper response explorer3hump_ei fix

Residual-vibration curves for every input shaper (ZV, MZV, ZVD, EI, 2hump/3hump EI), reproduced from shaper_defs.py. Overlays a corrected 3hump_ei against upstream for the coefficient bug, with an impulse view and a damping-ratio sweep showing where upstream's truncated polynomial degrades.

Smoothers vs shapersKalico feature

Kalico input smoothers replace a shaper's discrete impulses with a smooth polynomial window. Compare any shaper against any smoother on shared vibration-reduction axes and in the time domain, from the get_*_smoother definitions in shaper_defs.py.

Extruder smoother kernel fitfit rework

The polynomial kernel that keeps pressure advance synchronized with input shaping. Compares the fitted kernel before vs after the extruder_smoother.py rework (analytic targets, constrained least squares, Legendre basis) for every shaper — 3hump_ei improves the most.

Resonance test excitation explorerlimit fix

The motion TEST_RESONANCES generates — frequency sweep, per-cycle acceleration, and the optional sweeping oscillation. Overlays the enforced accel/velocity limits derived from the real sequence against upstream's fixed formula, which ignores the sweeping term.

Shaper estimation: old vs new~2x speedup

How SHAPER_CALIBRATE scores a shaper: the old finite-difference of the step response vs an analytic velocity with kink-exact evaluation and parabolic-refined minimum. Drop the sample count and watch the old method deviate while the new one stays exact.
Note on provenance. The pressure-advance page models a custom bleeding-edge-v2 implementation (tanh/recipr nonlinear PA) that is not in upstream Kalico or Klipper. The input-shaper, smoother, resonance-excitation, and estimation pages reproduce the Kalico shaper code (shaper_defs.py, resonance_tester.py, shaper_calibrate.py) with proposed fixes and contrast them against current upstream Kalico (bleeding-edge-v2).
